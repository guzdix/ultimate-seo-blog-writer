## Strict Dash and Hyphen Control

The final output must avoid unnecessary dash characters.

This rule applies to every part of the content, including:
- Title
- TL;DR
- Body content
- H2 headings
- H3 headings
- FAQ questions
- FAQ answers
- Meta titles
- Meta descriptions
- Schema text
- CTA sections
- Post delivery summary
- Tables
- Bullet points

### Characters to Avoid

Never use em dashes anywhere in the content.

Do not use this character:

—

Avoid en dashes unless they are truly required.

Do not use this character as punctuation:

–

Avoid decorative hyphens.

Do not use hyphens as sentence breaks, pauses, or stylistic separators.

### What to Use Instead

Instead of em dashes, en dashes, or decorative hyphens:

- Use commas, colons, or semicolons for pauses.
- Use parentheses for asides.
- Split into two sentences if connecting independent thoughts.
- Use "which" or "and" to restructure the sentence.

### Unnecessary Hyphenated Words

Avoid unnecessary hyphenated compound words.

Do not write phrases like:

- AI-powered
- SEO-friendly
- long-form
- source-backed
- user-focused
- high-quality
- step-by-step
- copy-paste
- rank-ready
- conversion-driven
- search-friendly
- data-driven
- content-led
- brand-focused
- beginner-friendly
- mobile-friendly
- user-first
- keyword-rich
- results-driven
- intent-based
- trust-building

Rewrite them naturally instead:

- AI-powered content → content powered by AI
- SEO-friendly article → article optimized for SEO
- long-form blog post → detailed blog post
- source-backed claims → claims supported by sources
- user-focused writing → writing focused on users
- high-quality content → quality content
- step-by-step guide → guide with clear steps
- copy-paste ready → ready to copy and paste
- rank-ready article → article prepared for ranking
- conversion-driven CTA → CTA designed for conversions
- search-friendly structure → structure optimized for search
- data-driven insight → insight based on data
- content-led strategy → strategy led by content
- brand-focused writing → writing focused on the brand
- beginner-friendly guide → guide for beginners
- mobile-friendly website → website that works well on mobile
- user-first content → content created for users first
- keyword-rich title → title with relevant keywords
- results-driven strategy → strategy focused on results
- intent-based outline → outline based on search intent
- trust-building details → details that build trust

### When Hyphens Are Allowed

Hyphens are allowed only when they are truly required.

Allowed cases include:

- URLs
- File names
- Code
- Commands
- Dates
- Negative numbers
- Schema values
- Official brand names
- Product names
- Technical terms that are normally written with hyphens
- Quoted source titles that already contain hyphens

Examples of allowed usage:

- `schema-generator.md`
- `ultimate-seo-blog-writer.zip`
- `2026-05-22`
- `-10`
- `application/ld+json`
- `x-robots-tag`
- `rel="nofollow"`
- `no-index` only if used as a technical SEO term

### Examples

Bad:

```text
This SEO-friendly, AI-powered, source-backed article helps create rank-ready content.
```

Good:

```text
This article is optimized for SEO, powered by AI, supported by sources, and prepared for ranking.
```

Bad:

```text
This guide explains SEO — from keyword research to content optimization.
```

Good:

```text
This guide explains SEO, from keyword research to content optimization.
```

Bad:

```text
This is a beginner-friendly, step-by-step guide.
```

Good:

```text
This is a guide for beginners with clear steps.
```

Bad:

```text
The result is high-quality content that is conversion-driven.
```

Good:

```text
The result is quality content designed for conversions.
```

### Final Output Rule

Before delivering any final content, scan the entire output.

Check for:

1. Em dashes
2. En dashes
3. Decorative hyphens
4. Unnecessary hyphenated compound words
5. Hyphens used as sentence breaks
6. Hyphens used as stylistic separators

If the final content contains unnecessary em dashes, en dashes, or hyphens, revise the content before showing it to the user.

The final output must prioritize natural phrasing without dash punctuation.

### Dash Quality Checklist

Before final delivery, verify:

- No em dash is used anywhere.
- No en dash is used unless truly required.
- No decorative hyphen is used as punctuation.
- No unnecessary hyphenated compound words are used.
- No hyphen is used as a sentence pause.
- No hyphen is used as a stylistic separator.
- Hyphens appear only in URLs, file names, code, dates, schema, brand names, product names, negative numbers, or technical terms where the hyphen is required.
